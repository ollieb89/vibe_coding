## Research & Analysis Commands

### `/sc:research` ‚Äì Deep Web Research with Adaptive Planning
**Complexity:** Advanced | **Category:** Research & Analysis

**Purpose:** Conduct comprehensive web research with intelligent planning and multi-hop reasoning

**Usage:**
```bash
/sc:research "[query]"
/sc:research "[query]" --depth quick|standard|deep|exhaustive
/sc:research "[query]" --strategy planning|intent|unified
/sc:research "[query]" --domains [domain1,domain2]
```

**Behavioral Flow:**
1. **Understand** ‚Äì Assess query complexity and ambiguity
2. **Plan** ‚Äì Select strategy based on complexity
3. **TodoWrite** ‚Äì Create adaptive task hierarchy
4. **Execute** ‚Äì Parallel searches and extraction
5. **Track** ‚Äì Monitor progress continuously
6. **Validate** ‚Äì Verify evidence and resolve contradictions

**Research Depth Levels:**

| Depth | Sources | Hops | Time | Best For |
|:-----:|:-------:|:----:|:----:|----------|
| **Quick** | 5-10 | 1 | ~2min | Simple facts |
| **Standard** | 10-20 | 3 | ~5min | General research |
| **Deep** | 20-40 | 4 | ~8min | Comprehensive |
| **Exhaustive** | 40+ | 5 | ~10min | Academic level |

**Key Patterns:**
- **Parallel Execution** ‚Äì Batch independent searches
- **Evidence Management** ‚Äì Track sources and confidence
- **Adaptive Depth** ‚Äì Adjust based on query complexity
- **Multi-Hop Exploration** ‚Äì Entity expansion and concept deepening

**MCP Integration:**
- **Tavily MCP** ‚Äì Primary search and discovery
- **Sequential MCP** ‚Äì Complex reasoning and synthesis
- **Playwright MCP** ‚Äì JavaScript-heavy content extraction
- **Serena MCP** ‚Äì Research persistence

**Examples:**
```bash
# Basic research with automatic depth
/sc:research "latest AI developments 2024"

# Controlled deep research
/sc:research "quantum computing breakthroughs" --depth deep

# Specific strategy selection
/sc:research "market analysis" --strategy planning-only

# Domain-filtered research
/sc:research "React patterns" --domains "reactjs.org,github.com"
```

## Behavioral Modes

SuperClaude operates in **7 adaptive behavioral modes** for different contexts:

### 1. **Brainstorming Mode**
- **Activation:** `/sc:brainstorm` or ambiguous requirements
- **Behavior:** Socratic dialogue, multi-persona analysis, specification generation
- **Best For:** Requirements discovery, feature planning

### 2. **Business Panel Mode**
- **Activation:** `/sc:business-panel` or strategic decisions
- **Behavior:** Multi-expert analysis, trade-off evaluation, strategic recommendations
- **Best For:** High-level decisions, strategic planning

### 3. **Deep Research Mode**
- **Activation:** `/sc:research` or knowledge gaps beyond cutoff
- **Behavior:** Autonomous planning, parallel searching, evidence synthesis
- **Best For:** Research, market analysis, competitive intelligence

### 4. **Orchestration Mode**
- **Activation:** `/sc:workflow`, `/sc:implement`, or complex projects
- **Behavior:** Efficient tool coordination, task distribution, progress tracking
- **Best For:** Multi-step projects, workflow generation

### 5. **Token-Efficiency Mode**
- **Activation:** Context limits or large codebases
- **Behavior:** Compression, focused analysis, direct recommendations
- **Best For:** Context management, large projects

### 6. **Task Management Mode**
- **Activation:** `/sc:task` or systematic organization
- **Behavior:** Breakdown, dependency mapping, progress tracking
- **Best For:** Complex task organization, execution planning

### 7. **Introspection Mode**
- **Activation:** `/sc:reflect` or meta-analysis
- **Behavior:** Self-analysis, pattern recognition, system evaluation
- **Best For:** Architecture review, decision validation

### Pattern 3: Code ‚Üí Test ‚Üí Improve

```bash
# 1. Implement feature
/sc:implement <specification>

# 2. Generate tests
/sc:test <implementation> --type all --coverage 80

# 3. Improve quality
/sc:improve <code> --focus quality
```

### Pattern 4: Complex Project Orchestration

```bash
# 1. Brainstorm requirements
/sc:brainstorm <project-vision>

# 2. Generate specification with business review
/sc:business-panel <brainstorm>

# 3. Create workflows for parallel execution
/sc:workflow <specification> --parallel

# 4. Spawn specialized agents
/sc:spawn frontend-specialist "Build UI components"
/sc:spawn backend-developer "Build APIs"
/sc:spawn security-engineer "Security review"
```

### Best Practices

1. **Start with `/sc:brainstorm`** for unclear requirements
2. **Use `/sc:business-panel`** for strategic decisions
3. **Leverage `/sc:research`** for knowledge beyond cutoff date
4. **Chain commands** for complex workflows (D‚ÜíP‚ÜíI pattern)
5. **Use `--parallel`** flag for independent work streams
6. **Save checkpoints** with `/sc:save` after major milestones
7. **Monitor background agents** with `/sc:spawn --monitor`
8. **Review specifications** before implementation
9. **Test thoroughly** with `/sc:test --type all`
10. **Document decisions** in `/sc:save` for team learning
